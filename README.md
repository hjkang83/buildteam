# Data 기반 Multi-Agent 부동산 투자 자문 시스템 — AI 부동산 투자 자문 시스템

> "강남 오피스텔 수익률 3%면 낮은 거 아냐?" 한마디에,
> CFO·CSO·고문 세 명의 C-suite가 실거래 데이터를 놓고 토론합니다.

\---

## 한 줄 요약

**공공 부동산 데이터 기반으로, 3명의 AI C-suite 에이전트가 텍스트로 토론하며 투자 의사결정을 돕는 멀티 에이전트 자문 시스템**

\---

## 이 프로젝트가 해결하는 문제

부동산 투자를 결정할 때, 혼자 고민하면 확증 편향에 빠진다. "좋아 보이는 정보"만 찾게 되고, 반론을 줄 사람이 없다. 전문가를 고용하자니 수천만 원이 들고, 주변에 물어봐도 데이터 기반 분석은 해줄 수 없다.

ChatGPT에 물어볼 수 있지만, 단일 AI는 하나의 시각만 제시한다. 그리고 근거 없이 숫자를 말하는 AI는 위험하다.

**말하는 비서실**은 이 문제를 푼다. 실거래가 데이터를 근거로 CFO·CSO·고문이 각자의 전문 영역에서 토론하며, 모든 숫자에 출처를 붙인다.

\---

## 타겟 사용자

**부동산 투자를 검토하는 1인 창업가 / 소규모 투자자**

* 매물을 봤는데 "이거 괜찮은 건가?" 판단이 서지 않는 사람
* 주변에 같이 분석해줄 전문가가 없는 사람
* ChatGPT에 물어봤는데 "출처 없는 긍정적 답변"만 받아본 사람

\---

## 핵심 기능

### 1\. 세 명의 C-suite가 토론한다

|에이전트|직함|전문 영역|응답 예시|
|-|-|-|-|
|A|**CFO** (재무총괄)|수익률, 현금 흐름, 세금, 대출|"대출 60% 끼면 레버리지 수익률 4.2%인데, 취득세 떼면 첫해 마이너스입니다"|
|B|**CSO** (전략총괄)|시장 분석, 타이밍, 리스크|"고금리 국면에서 오피스텔 진입은 타이밍이 안 맞습니다. 6개월 뒤 재검토가 맞아요"|
|C|**CHO/고문** (자문역)|투자 철학, 장기 목표|"월세가 목적이라면 강남 타이틀보다 현금 흐름을 우선해야 합니다"|
|서기|**비서실장**|회의 정리|"오늘 결정: 강남 보류, 성수 선회. 내일까지 급매물 3건 리스트 준비"|

**왜 C-suite인가?**

* 일반적인 "찬성/반대/조언" 구분보다 **각자의 책임 영역이 명확**하다
* CFO는 숫자에 대해서만, CSO는 전략에 대해서만, 고문은 큰 그림에 대해서만 발언한다
* 실제 기업의 의사결정 구조를 그대로 반영하여 분석의 빈틈이 줄어든다

### 2\. 모든 숫자에 출처가 있다

에이전트가 "수익률 4.2%"라고 말하면, 반드시 근거를 명시한다:

* "국토교통부 실거래가 API 기준, 2025년 1\~3월 해당 단지 거래가"
* "네이버 부동산 호가 기준, 2026.04 기준"

근거 없이 숫자를 말하는 AI는 위험하다. **출처를 보여주는 AI는 신뢰할 수 있다.**

### 3\. 파일 업로드로 내 상황을 분석한다

* Excel: 보유 부동산 현황, 수익률 비교표
* PDF: 등기부등본, 건축물대장
* 텍스트: 매물 정보 복사-붙여넣기

파일을 올리면 CFO는 재무 관점에서, CSO는 전략 관점에서, 고문은 장기 방향성 관점에서 각각 분석해준다.

### 4\. 비서실장이 회의록을 정리한다

대화가 끝나면 서기(비서실장) 에이전트가 자동 정리:

* 핵심 결정사항 (예: "강남 보류, 성수 선회")
* 넥스트 액션 (예: "성수 신축 급매물 3건 리스트 확보")
* 각 C-suite의 핵심 의견 요약

## 왜 텍스트 기반인가? (음성이 아닌 이유)

부동산 의사결정은 데이터 집약적이다:

* 수익률 "3.2% vs 4.1% vs 2.8%"를 귀로 듣고 비교하기 어렵다
* 이전 발언을 다시 확인하려면 스크롤이 되감기보다 빠르다
* Excel/PDF 첨부는 텍스트 채팅에서만 자연스럽다

텍스트 기반으로 전환하면 기술적으로도 단순해진다:

* STT, TTS, Pipecat, 지연 마스킹, 중단 로직이 전부 불필요
* 비용이 LLM 호출 비용(시간당 150\~3,500원)만 남는다
* 작은 팀에서 구현 가능한 범위 안에 들어온다

* 모든 숫자에 출처 명시

## MVP 범위 (5분짜리 데모)

**시나리오: "강남 오피스텔 투자 검토"**

사용자가 매물 정보를 입력하면:

1. CFO가 수익률·대출·세금 분석
2. CSO가 시장 타이밍·리스크 지적
3. 고문이 투자 목적 대비 적합성 조언
4. 비서실장이 결론 요약 + 넥스트 액션 도출

**이것만 end-to-end로 작동하게 만드는 것이 목표.**
