# 부동산 투자 자문 시스템 용어 사전

> Premortem 시나리오 7 ("동상이몽") 방지를 위한 공용 용어 정의.
> 팀 전원이 같은 단어를 같은 뜻으로 쓰도록 한다.

---

## 🧑‍🤝‍🧑 에이전트 (Agent)

**정의**: 각각 고유한 페르소나(역할/말투/금지행동)를 가진 LLM 인스턴스.

- **CFO** (재무총괄): 수익률·세금·대출·현금흐름 전담
- **CSO** (전략총괄): 타이밍·입지·리스크·경쟁 전담
- **고문** (자문역): 투자 목적·철학·장기 방향 전담
- **비서실장** (서기): 중립적 회의록 정리 + 넥스트 액션 도출
- 각 에이전트는 `agents/*.md`에 명세된 규칙을 따른다.

---

## 📊 출처 명시 (Source Citation)

**정의**: CFO가 수치를 말할 때 반드시 데이터 근거를 밝히는 규칙.

- 형식: `[출처: 국토교통부 실거래가 API, 2025 Q1]`
- RAG 검색 결과에 없으면 "이 부분은 데이터가 부족합니다"로 답할 것
- **핵심 가치 1번** ("출처 있는 숫자만 말한다")의 실천 기준

---

## 🏢 C-suite

**정의**: 최고 경영진(Chief ~~ Officer) 직함 체계. 우리 시스템에서는 각 에이전트의 **책임 영역을 분리**하기 위해 사용.

- CFO ≠ "숫자만 아는 사람" → 재무적 검증 전담
- CSO ≠ "반대만 하는 사람" → 전략적 리스크 분석 전담
- 고문 ≠ "철학만 하는 사람" → 투자 목적 정합성 점검 전담

---

## 🔴 영역 경계 (Role Boundary)

**정의**: 각 C-suite가 자기 전문 영역에서만 발언하도록 강제하는 규칙.

| 발언 OK | 발언 금지 |
|---------|----------|
| CFO → 수익률, 세금, 대출 | CFO → 시장 전망, 인생 조언 |
| CSO → 타이밍, 리스크, 입지 | CSO → 세금 계산, 투자 철학 |
| 고문 → 투자 목적, 명저 | 고문 → 수치 계산, 시장 예측 |

---

## ✅ 넥스트 액션 (Next Action)

**정의**: 회의 결과로 도출되는 **동사로 시작하는 구체 행동**. 담당자와 기한이 반드시 붙는다.

- ❌ "알아보기"
- ✅ "국토교통부 실거래가 API로 해당 단지 3년 거래 내역 조회하기 (담당: 대표님, 기한: 4/24)"

---

## 📝 회의록 (Meeting Minutes)

**정의**: 비서실장 에이전트가 **고정 템플릿**에 따라 자동 생성하는 Markdown 문서.

- 필수 섹션: 핵심 안건 / 주요 논의(C-suite별 1줄) / 결정사항 / 보류사항 / 넥스트 액션 / 다음 회의 아젠다
- `agents/clerk.md`의 템플릿이 유일한 진실원(Source of Truth)
- 파일 위치: `meetings/{YYYY-MM-DD}-{slug}.md`

---

## 🎯 Gold Standard

**정의**: "이 질문이 들어오면 이 응답이 나와야 한다"는 **기준 응답**. 각 페르소나 명세서에 예시가 포함되어 있고, 프롬프트 튜닝의 검증 기준이 된다.

- 데모 시나리오: **강남 오피스텔 투자 검토**

---

## 🔴 반론 강제 (Forced Objection)

**정의**: CSO 에이전트가 **모든 응답에 반드시 리스크/반론/의문을 1개 이상 포함**하도록 강제하는 규칙.

- "동의합니다"만 하면 = CSO의 실패
- 세 C-suite가 전부 같은 결론이면 → "빠뜨린 리스크는?" 자동 추가 질문

---

## 📡 실거래가 API

**정의**: 국토교통부가 제공하는 부동산 실거래 가격 정보 공공 API.

- 아파트, 오피스텔, 연립다세대, 단독다가구 실거래가
- 접근: [공공데이터포털](https://www.data.go.kr/) API 키 발급
- CFO의 핵심 데이터 소스

---

## 🔄 RAG (Retrieval-Augmented Generation)

**정의**: 질문 전에 관련 데이터(실거래가, 명저 인용)를 검색해서 LLM 컨텍스트에 주입하는 기법.

- 부동산 데이터 RAG: 실거래가, 시세, 정책 → CFO·CSO용
- 명저 RAG: 부동산 투자 명저 텍스트 → 고문용
- 저장소: Pinecone 또는 ChromaDB

---

## 🏷 MVP 정의

**"강남 오피스텔 투자 검토" 시나리오가 end-to-end로 작동하는 5분 데모.**

1. 사용자 입력 → 2. 데이터 조회 → 3. CFO·CSO·고문 토론 → 4. 비서실장 요약

이 흐름이 끊기면 MVP로 인정하지 않는다.
