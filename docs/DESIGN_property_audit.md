## Problem Statement

원래 ask는 "일반 사용자도 매물 비교를 통해 쉽게 투자 의사결정"이었으나, Q1·Q3 진단을 통해 **진짜 wedge는 'A vs B 비교'가 아닌 '단일 매물 호가 적정성 절대평가'**로 reframe됨. 사용자가 막힌 지점은 "어느 매물이 더 좋은가"가 아니라 "이 매물의 호가가 가치 대비 정당한가"였음.

## Demand Evidence (N=2 동일 패턴)

| | 본인 | 강희준 (44세, 통신사 16년차) |
|---|---|---|
| 지역·매물 | 노원구 아파트 | 노원 청구3차 + 롯데우성 |
| 도구 | 호갱노노 등 | (유사) |
| 막힘 지점 | "이 가격의 가치가 있는지 모름" | "호가 과대평가 같음" |
| 결과 | 미룸 | 미룸 |

두 사례 모두 단일 매물의 절대 가치 평가에서 막혀 결정 보류. 이는 약한 의미의 "관심"이 아닌 행동 기반 demand evidence (실제 검토했으나 결정 못 내림).

## Status Quo (진짜 경쟁자)

Q2 답변: 일반 사용자는 이 막힘을 **부동산 중개인 의견에 의존**해 해결한다. 시장은 이미 "자문에 비용 지불"하는 상태. 다만 중개인은 수수료 구조상 "사세요" 편향이 있어 **객관적 second opinion이 부재**. 이 시장 갭이 차별점의 근거.

## Target User & Narrowest Wedge

**1차 타겟**: **B 아키타입 + 일반인 둘 다** — 단일 데이터·로직, 출력 UI 이중 모드로 분기.

| 모드 | 대상 | 출력 |
|---|---|---|
| **단순(Simple)** | 일반인 (부동산 투자 입문자) | "이 호가는 적정·고평가·저평가입니다" + 근거 3줄 (수치는 압축, 친절 해석) |
| **심화(Pro)** | B 아키타입 (강희준 같은 사람) | P5/P50/P95 적정가 분포 + 페르소나 3인 토론 전문 + 수익률·세후 순현금 |

기본은 단순 모드, "심화 보기" 클릭으로 페르소나 토론 펼침.

**Wedge**: 주소 + 평형 + 호가 입력 → 단순 모드 1페이지 (기본) → 심화 토글 시 전문 출력.

## Premises

| | 가설 | 합의 상태 |
|---|---|---|
| P1 | Wedge는 'A vs B 비교'가 아닌 '단일 매물 절대평가'다 | 미체크지만 Q4-A 선택으로 함의 확정 |
| P2 | 진짜 경쟁자는 ChatGPT가 아닌 부동산 중개인이다 | **OPEN — 사용자 인터뷰로 검증 필요** (시뮬레이션상 framing 후보: "인센티브 구조 차이 — 중개인은 거래성사가 수수료, buildteam은 결정 정확성이 가치") |
| P3 | 차별점은 "중개인 편향에서 자유로운 근거 있는 second opinion"이다 | ✅ 확정 |
| P4 | 1차 타겟은 B 아키타입 + 일반인 둘 다, 출력 UI를 단순·심화로 분기 | ✅ 확정 (재검토 후) — **단, 일반인 모드는 "매물 후보 좁히기" pre-step 추가 필요 (시뮬 통찰)** |

## Approaches Considered

### A. Minimal Viable: 1페이지 호가 평가
- **Summary**: 주소·평형·호가 입력 → 동일 단지 6개월 실거래가 평균 + 수익률 → 1페이지
- **Effort**: S (3-5일)
- **Risk**: Low
- **Reuses**: real_estate.py + yield_analyzer.py
- **Pros**: N=2 직결, 자산 100% 재활용, 해커톤 시간 적합
- **Cons**: "적정가" 정의가 단순 평균이라 비판 가능, 페르소나 미사용 → buildteam 차별점 약함

### B. Ideal: 페르소나 3인 토론 기반 평가 ✅ **선택됨**
- **Summary**: buildteam 전체 파이프라인 + CFO·CSO·투자컨설턴트 3인 토론 → 비서실장 1페이지
- **Effort**: M-L (1-3주)
- **Risk**: Med
- **Reuses**: 전체 buildteam infrastructure
- **Pros**: 전체 가치 활용, P3 차별점 직접 구현, 페르소나가 객관성 보강
- **Cons**: LLM 비용·지연 (5초+), 일반인이 토론 결과 이해 어려울 수 있음

### C. Creative: '중개인 시뮬레이션' 이중 페르소나
- **Summary**: 가상 "동네 중개인 김씨"(편향) + "투자컨설턴트"(근거 기반) 의견 동시 제시
- **Effort**: M
- **Risk**: Med-High
- **Reuses**: 페르소나 시스템
- **Pros**: P3 차별점 시각화 강력, 발표용 데모 임팩트
- **Cons**: caricature 위험, 실제 중개인과 평판 갈등 가능

## Recommended Approach: B

## Success Criteria

- **Binary (B)**: 강희준이 노원 청구3차 호가에 대해 **심화 모드** 출력을 보고 "이 가격에 사야/말지" 결정 내릴 수 있는가?
- **Binary (일반인)**: 부동산 입문자 1명이 **단순 모드** 출력만 보고 동일 결정을 내릴 수 있는가?
- **N=5 인터뷰 (B 3명 + 일반인 2명)**:
  - B: "심화 모드가 중개인 의견보다 신뢰 가는가?" — 2/3 이상 Y → P3 확정
  - 일반인: "단순 모드 라벨이 결정에 도움 됐는가?" — 1/2 이상 Y → 단순 모드 유효성 확정
- **시연**: 캡스톤 발표 5분에 단순→심화 토글 시연이 자연스럽게 들어가는가?

## The Assignment (시뮬 후 갱신)

**다음 1주일 안에 두 가지 트랙 병행:**

### 트랙 1 — 도메인 무장 (founder 본인)
- 한국감정원 부동산통계정보시스템 R-ONE 핵심 보고서 3개 정독
- 국토부 실거래가 공개 시스템 + KB부동산 시계열 데이터 구조 학습
- 헤도닉 회귀 모델 기본 (층/향/리모델링 계수가 학술적으로 어떻게 추정되는지) 1편 논문 + 1개 블로그 정독
- 목표: 인터뷰에서 "그 0.7%는 어디서 나왔어요?" 즉답 가능

### 트랙 2 — 실제 인터뷰 (강희준 + 일반인 1명)

1. **강희준 (B 아키타입)** — 노원 청구3차 호가에 대해 buildteam `demo_mock` 출력 보여주기
   - "이 출력 형태로 호가 적정성을 받았다면 결정에 도움 됐을지" 묻기
   - **추적 신호**: ① "중개인 말과 어떻게 다른가" 자발 비교 (P2) ② "월 X원이면 쓸 것 같다" pricing 신호 ③ "이 매물 다른 거도 돌려보자" repeat 신호 ④ **"결정이 무서운 게 풀렸나"** (P5)

2. **일반인 1명 (매물 미정 상태)** — 가상 매물 1건 단순 모드 출력만 보여주기
   - "이 한 줄 라벨 + 근거 3줄로 결정에 도움 됐을지" 묻기
   - **추적 신호**: ① 추가 정보 욕구 (심화 모드 유효성) ② **"빨리 결정해야 한다는 압박이 풀렸나"** (P5) ③ "그냥 호갱노노로도 되겠는데"라고 하면 P3 차별점 약함

3. 두 인터뷰 결과를 비교 정리 → 다음 office-hours 또는 plan-eng-review 입력

핵심: P5 ("결정 압박 해소") 신호를 두 인터뷰에서 적극 듣기. 만약 강하면 wedge를 "호가 적정성 평가"에서 "결정 막힘 해소"로 reframe해야 할 수 있음.

## What I noticed about how you think

- **Agency 신호**: 이미 본인이 노원 검토했고, 호갱노노 한계를 직접 부딪쳤음. "한 번 막혔다"가 아니라 "결정 미루는 게 패턴"임을 인지.
- **Taste over expedience**: 권장 옵션 A(빠른 prototype)를 거부하고 B(ideal architecture)를 선택. 시간보다 차별점 우선.
- **Conviction, not compliance**: 4개 premise 중 1개만 동의 (P3). 무비판 동의보다 훨씬 좋은 founder signal.
- **시뮬에서 솔직함**: "도메인 지식이 없어서 힘드네"라고 말씀하신 거 — founder 자가진단 강함. 모르는 걸 모른다고 하는 게 좋은 신호. 약한 founder는 그걸 가립니다.
- **다만 한 가지**: P1·P2·P4 거절 이유를 한 줄로 articulate 하지 않았음. 시뮬에서도 즉답 회피 패턴(첫 번째 강희준 답변에서 feature dump). 다음에 거절·답변 이유를 한 줄로 articulate하는 연습이 design doc 다듬기와 인터뷰 둘 다에 필요.

## Simulation Insights (Round 1: B 강희준 + Round 2: 일반 김지영)

핵심 통찰 4개:

1. **공통 needs**: 두 페르소나 모두 "결정 압박 해소 / 확신"을 더 강하게 표현. wedge "호가 적정성"보다 깊은 needs 가능성 → P5로 등록
2. **분기점**: 강희준은 매물 있음 (verdict 즉시 가능) / 김지영은 매물 없음 (지역·예산 좁히기 → 매물 후보 → verdict 3단계 필요)
3. **단순 모드 한계**: 단순 모드 1페이지 출력 자체는 작동 가능. 다만 일반인은 매물 입력까지 못 오는 경우가 많음 → pre-step 또는 명시적 범위 한정 필요
4. **도메인 무장 필요**: 헤도닉 계수, 분포 통계 표본 크기, "친구 말 통계 거짓" 같은 답변은 즉시 출처 필요. MANIFESTO 1번 적용
