## 줄기 1: 재무적으로 검증해야 한다 → CFO

### 위에서 아래로 읽기

```
혼자서도 근거 있는 투자 판단을 내릴 수 있다
    ↓ 그러려면?
재무적으로 검증해야 한다 (수익률, 세금, 현금 흐름)
    ↓ 왜 지금은 안 되나?
수익률 계산, 취득세/양도세, 대출 레버리지 효과를 
정확히 계산하려면 전문 지식이 필요하다.
ChatGPT에 물어봐도 "대략적으로" 말할 뿐 
실거래가 기반 정확한 계산은 안 해준다.
    ↓ 그래서 필요한 것은?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
\[기능] CFO 에이전트
 - 국토교통부 실거래가 API 연동
 - 수익률/레버리지/세금 자동 계산
 - 모든 수치에 출처 명시
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

|판단|결론|
|-|-|
|이 기능 없으면?|근거 없는 숫자를 말하게 됨 → 핵심 가치 위반|
|중요도|**필수**|

## 줄기 2: 전략적으로 검증해야 한다 → CSO

### 위에서 아래로 읽기

```
혼자서도 근거 있는 투자 판단을 내릴 수 있다
    ↓ 그러려면?
전략적으로 검증해야 한다 (타이밍, 리스크, 시장 흐름)
    ↓ 왜 지금은 안 되나?
혼자 투자를 검토하면 "좋아 보이는 정보"만 찾는다.
반론을 줄 사람이 없으면 확증 편향이 굳어진다.
전문가 고용은 수천만 원. 주변은 데이터 기반 분석 불가.
    ↓ 그래서 필요한 것은?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
\[기능] CSO 에이전트
 - 시장 데이터 기반 타이밍 분석
 - 반드시 1개 이상 리스크/반론 제기
 - 시나리오별 비교 분석
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

|판단|결론|
|-|-|
|이 기능 없으면?|확증 편향 해결 안 됨 → 단일 AI와 동일|
|중요도|**필수** — 차별화의 핵심|

## 줄기 3: 나에게 맞는 투자인지 확인 → 투자컨설턴트

### 위에서 아래로 읽기

```
혼자서도 근거 있는 투자 판단을 내릴 수 있다
    ↓ 그러려면?
숫자가 맞아도 내 상황에 안 맞으면 잘못된 투자다
    ↓ 왜 지금은 안 되나?
CFO와 CSO가 "숫자상 괜찮다"고 해도,
내 리스크 허용도, 포트폴리오 상황, 투자 유형과
맞는지 판단해줄 전문가가 없다.
대안 투자와 비교하거나 포트폴리오 관점에서
적합성을 평가해줄 사람이 필요하다.
    ↓ 그래서 필요한 것은?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
\[기능] 투자컨설턴트 에이전트
 - 클라이언트 상황 기반 투자 적합성 평가
 - 대안 투자 비교 및 포트폴리오 자문
 - 리스크 프로파일 매칭 + 출구 전략 검토
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

|판단|결론|
|-|-|
|이 기능 없으면?|숫자는 맞지만 내 상황에 안 맞는 투자를 할 수 있음|
|중요도|**높음** — 다른 서비스에 없는 차별점|

## 결론: C-suite 구조가 Why Tree를 더 선명하게 만든다

기존의 "실무/레드팀/멘토"는 역할 구분이 감각적이었다. C-suite로 바꾸니:

* **CFO** → "재무 검증"이라는 명확한 줄기에 연결
* **CSO** → "전략 검증 + 확증 편향 방지"에 연결
* **투자컨설턴트** → "클라이언트 맞춤 투자 적합성 평가"에 연결

기능을 추가하고 싶을 때의 질문:

> "이 기능에서 위로 올라갔을 때, 세 줄기 중 하나를 거쳐 핵심 가치에 도달하는가?"

도달하면 만든다. 도달하지 않으면 만들지 않는다.
