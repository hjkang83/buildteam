# 말하는 비서실 Why Tree
> 4/11 팀원들과의 Why Tree 논의를 기반으로 정리
> "왜 이 서비스가 필요한가?"를 뿌리까지 파고든 결과

---

## Why Tree란?

사용자의 니즈에 대해 "왜 그걸 원하는가?"를 반복적으로 물어서 근본 욕구를 찾고, 각 욕구를 충족시킬 해결 수단을 도출하는 분석 방법이다.

---

## 최상위 문제

> **"혼자 의사결정해야 하는 리더/창업가가, 중요한 결정 앞에서 막막하고 고립되어 있다"**

---

## Why Tree 구조

```
혼자서 결정해야 하는데 막막하다
│
├── 1. 같이 의논할 사람이 없다
├── 2. AI한테 물어보는 것도 일이다
└── 3. 결정하고 나면 흐지부지된다
```

---

## 줄기 1: 같이 의논할 사람이 없다

```
같이 의논할 사람이 없다
│
├─ WHY? 전문가를 고용하기엔 비용이 너무 크다
│  │
│  └─ WHY? 수천만 원의 인건비 + 채용/교육이라는 HR 리스크
│          (비서진을 두는 건 대기업 회장님의 특권이었다)
│
├─ WHY? 혼자 결정하면 편향에 빠진다
│  │
│  └─ WHY? 내 생각에 동의해주는 정보만 찾게 된다
│          (확증 편향. 반대 의견을 줄 사람이 없다)
│
└─ WHY? 주변에 물어봐도 전문성이 부족하다
   │
   └─ WHY? 친구, 가족은 응원은 해주지만
            데이터 기반 분석은 못 해준다
```

### 인사이트

리더가 진짜 원하는 건 "답"이 아니라 **"다른 관점에서 반론을 해주고, 내 생각의 허점을 찔러줄 팀"**이다. 대기업 회장님이 비서진에게 가진 것을 AI로 민주화하는 것이 핵심이다.

### 말하는 비서실의 대응

| 기존 방법 | 문제점 | 우리의 해결 |
|-----------|--------|------------|
| 전문가/비서 고용 | 수천만 원 인건비, HR 리스크 | AI 에이전트 팀 = 월 몇 만원 (시간당 1,210원~) |
| 혼자 고민 | 확증 편향, 예스맨 함정 | 레드팀 에이전트가 반드시 반론 제기 |
| 주변에 물어보기 | 전문성 부족 | 명저 20권 + 실시간 데이터 학습한 멘토 |

---

## 줄기 2: AI한테 물어보는 것도 일이다

```
AI한테 물어보는 것도 일이다
│
├─ WHY? 프롬프트를 짜는 것 자체가 노동이다
│  │
│  └─ WHY? 정제된 텍스트를 작성하려면 고도의 의지 에너지가 필요
│          (앉아서 치고, 먼저 물어보고, 맥락을 설명하고...)
│
├─ WHY? AI는 한 방향으로만 답한다
│  │
│  └─ WHY? 한 명의 AI에게 물으면 하나의 시각만 나온다
│          (찬성/반대/조언을 동시에 받을 수 없다)
│
└─ WHY? 대화가 아니라 질의응답이다
   │
   └─ WHY? 실제로 사람들이 일하는 방식은 "회의"인데,
            AI 서비스는 "검색창"에 머물러 있다
```

### 인사이트

사람들이 일하는 자연스러운 방식은 **"대화"**다. 회의하고, 보고하고, 토론한다. 하지만 지금의 AI 서비스는 "글을 써서 질문하기"라는 부자연스러운 인터페이스에 갇혀 있다. **의지의 문턱을 낮추려면 대화로 바꿔야 한다.**

### 말하는 비서실의 대응

| 기존 방법 | 문제점 | 우리의 해결 |
|-----------|--------|------------|
| ChatGPT 텍스트 입력 | 프롬프트 작성이 노동 | 말하기만 하면 됨 (STT 처리) |
| 단일 AI 챗봇 | 한 방향 시각 | 실무/레드팀/멘토 3인이 동시 토론 |
| 일회성 질의응답 | 맥락 유지 안 됨 | 역질문 + 추임새로 실제 회의처럼 티키타카 |

---

## 줄기 3: 결정하고 나면 흐지부지된다

```
결정하고 나면 흐지부지된다
│
├─ WHY? 회의 내용을 기록하지 않으니 까먹는다
│  │
│  └─ WHY? 기록하는 것도 별도의 노력이 필요
│          (회의하면서 동시에 메모하기 어렵다)
│
├─ WHY? 결정은 했는데 다음 할 일이 불명확하다
│  │
│  └─ WHY? "다음에 알아보자"로 끝나면 다음은 없다
│
└─ WHY? 지난 회의와 오늘 회의가 연결되지 않는다
   │
   └─ WHY? 맥락이 끊기면 매번 처음부터 다시 설명해야 한다
```

### 인사이트

의사결정의 가치는 결정 자체가 아니라 **"실행으로 이어지는 것"**에 있다. 대화로 끝나면 그냥 수다다. 회의록 → 넥스트 액션 → 다음 회의 아젠다로 이어지는 선순환이 있어야 한다.

### 말하는 비서실의 대응

| 기존 방법 | 문제점 | 우리의 해결 |
|-----------|--------|------------|
| 수동 메모/녹음 | 따로 정리해야 함 | 서기 에이전트가 실시간 스크립트 자동 생성 |
| "나중에 해야지" | 넥스트 액션 불명확 | 회의 종료 시 액션 플랜 자동 도출 |
| 매번 처음부터 설명 | 맥락 단절 | 옵시디언 저장 → RAG로 다음 회의에 자동 로드 |

## Why Tree → 핵심 질문 매핑

PPT의 평가 기준(4/11 Why Tree 논의 결과)과 연결하면:

```
WHY TREE 줄기                    →  평가 기준과의 연결
──────────────────────────────────────────────────────
같이 의논할 사람이 없다           →  Is AI-Native?
                                    수천만 원 인건비를 AI로 월 몇 만원에 해결

AI한테 물어보는 것도 일이다       →  So What? & What's New?
                                    의지 문턱을 낮추고, "대화로 일하는" 새로운 방식

결정하고 나면 흐지부지된다        →  Why You?
                                    1년 반 STT/LLM 경험으로 이 문제를 직접 느꼈다
```

## Why Tree → 기능 매핑 요약

```
WHY TREE 니즈                    →  말하는 비서실 기능
──────────────────────────────────────────────────────
전문가 고용이 너무 비싸다         →  🤖 AI 전문가 에이전트 팀 (시간당 1,210원~)
확증 편향에 빠진다               →  🔴 레드팀 에이전트 (독립적 반론)
프롬프트 짜기가 노동이다          →  🎙️ 음성 기반 인터페이스 (STT)
한 방향 시각만 나온다            →  👥 실무/레드팀/멘토 3인 동시 토론
회의 기록이 안 남는다            →  📝 서기 에이전트 (실시간 스크립트)
다음 할 일이 흐지부지            →  ✅ 넥스트 액션 플랜 자동 생성
맥락이 매번 끊긴다              →  🔄 옵시디언 저장 + RAG 선순환
```

---

## 결론

Why Tree 분석을 통해 확인한 것: 말하는 비서실의 모든 기능은 **리더의 실제 고충**에서 출발한다.

- **같이 의논할 사람이 없다** → 멀티 에이전트 팀
- **AI한테 물어보는 것도 일이다** → 음성 인터페이스
- **결정 후 흐지부지된다** → 회의록 + 넥스트 액션 자동화

기능을 추가하거나 뺄 때는 항상 이 세 줄기로 돌아와서, "이 기능이 어디에 해당하는가?"를 확인한다. 어디에도 해당하지 않으면 빼야 할 기능이다.
